# Proyecto: E-commerce Automation 
Este caso práctico se desarrolla de forma incremental a lo largo del curso. 
Los  alumnos  aplicarán  en  cada  etapa  los  conocimientos  adquiridos  en  los  módulos 
anteriores, construyendo progresivamente una automatización completa de un ﬂujo de 
e-commerce. 
Cada entrega parcial permite practicar, validar y consolidar el aprendizaje, hasta llegar 
a una implementación ﬁnal completa con reportes profesionales. 

## Entrega 1 — Análisis del Caso de Negocio 
Momento de entrega: Al ﬁnalizar Módulo 2 
### Objetivo 
Conﬁgurar el entorno profesional y lograr la primera interacción con el navegador 
### Actividades a realizar 
1.  Selección de sitio de pruebas 
Elegir un sitio web de pruebas para implementar la automatización. 
Ejemplos sugeridos: 
●  saucedemo.com 
 
2.  Análisis del ﬂujo completo del sistema 
Identiﬁcar las etapas del ﬂujo de compra: 
●  Login 
●  Selección de artículo 
●  Agregar al carrito 
●  Checkout 
●  Logout 
 
3.  Identiﬁcación de casos de prueba, páginas y elementos. 
●  Deﬁnir los casos de prueba principales del ﬂujo. 
●  Identiﬁcar las páginas involucradas en el ﬂujo. 
●  Localizar los elementos clave que serán automatizados. 
 
4.  Automatización inicial de pruebas. 
●  Preparación del proyecto inicial (dependencias, drivers, etc). 
●  Identiﬁcar validaciones críticas, esperas de selenium y su implementación. 
●  Automatización de los casos de prueba deﬁnidos. 

### Entregable   

Documento de análisis que incluya: 
●  Diagrama de ﬂujo del proceso de compra 
●  Listado de páginas identiﬁcadas 
●  Listado de elementos clave por página 
●  Plan de pruebas propuesto 
●  Proyecto Automation con casos de pruebas automatizadas. 

## Entrega  2  —  Implementación  de  Automatización 
con Page Object Model (POM) 
Momento de entrega:  Al ﬁnalizar Módulo 4 

### Objetivo 
Implementar la automatización del ﬂujo de e-commerce utilizando el patrón Page Object 
Model  (POM),  aplicando  buenas  prácticas  de  diseño,  reutilización  de  código  y 
organización del proyecto. 

### Actividades a realizar 
1. Creación de la estructura del proyecto 
●  Crear proyecto Maven  
●  Deﬁnir la estructura de carpetas para el patrón POM 
●  Conﬁgurar dependencias necesarias (Selenium, JUnit) 

2. Implementación de Page Objects principales 
LoginPage 
Elementos 
●  Campo usuario 
●  Campo contraseña 
●  Botón login 
 
Métodos 
●  login() 
●  validarMensajeError() 
 
HomePage / ProductPage 
Elementos 
●  Lista de productos 
●  Botones "Agregar al carrito" 
 
Métodos 
●  seleccionarProducto() 
●  agregarAlCarrito() 
 
CartPage 
Elementos 
●  Items del carrito 
●  Botón checkout 
 
Métodos 
●  veriﬁcarProductoEnCarrito() 
●  irACheckout() 
 
CheckoutPage 
Elementos 
●  Formulario de datos del cliente 
●  Botón ﬁnalizar compra 
 
Métodos 
●  completarDatos() 
●  ﬁnalizarCompra() 
 
### 3. Implementación de BasePage 
Centralizar funcionalidades comunes del sistema. 
Elementos comunes 
●  Header 
●  Footer 
●  Navegación 
Métodos comunes 
●  logout() 
●  navegar() 
 
### 4. Aplicación de Page Factory 
●  Implementar Page Factory para inicializar elementos. 
●  Refactorizar las Page Objects para aplicar esta técnica. 

### 5. Implementación de pruebas automatizadas 
Crear pruebas automatizadas con JUnit 5 que validen: 
●  Login exitoso 
●  Agregar producto al carrito 
●  Veriﬁcación del producto en carrito 
●  Flujo completo de compra 
 
Entregable   
Repositorio o código fuente con: 
●  Proyecto Maven conﬁgurado 
●  Implementación completa de Page Objects 
●  Uso de Page Factory 
●  Implementación de BasePage 
●  Pruebas automatizadas con JUnit 5 
●  Validación del ﬂujo completo de compra
